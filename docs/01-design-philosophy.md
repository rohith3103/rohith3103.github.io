# ROHITH.OS

Version: 2.0

Codename:
ROHITH.OS

Authoritative Product Specification

===========================================================
MISSION
===========================================================

You are NOT redesigning a portfolio.

You are engineering an interactive software product that
happens to represent my professional career.

Visitors should feel they are exploring an intelligent
engineering platform rather than scrolling through a resume.

The portfolio must communicate one clear message:

"I build intelligent software systems from AI models to production."

Every interaction must reinforce that message.

Never sacrifice usability for visual effects.

Never sacrifice accessibility for aesthetics.

Never sacrifice performance for animation.

Every animation must have meaning.

Every section must tell a story.

Every click must reward curiosity.

===========================================================
PRIMARY GOALS
===========================================================

The portfolio should achieve the following:

✓ memorable
✓ premium
✓ interactive
✓ elegant
✓ technically impressive
✓ recruiter friendly
✓ engineering focused
✓ AI focused
✓ DevOps focused
✓ accessible
✓ responsive
✓ production ready

The visitor should remember the portfolio days after leaving.

The reaction should be:

"I've never seen someone explain engineering this well."

NOT

"Cool animations."

===========================================================
TARGET AUDIENCE
===========================================================

The experience must satisfy three completely different visitors.

-----------------------------------------------------------

1. Recruiter

Needs:

• understand candidate in under 60 seconds
• resume
• contact
• skills
• projects
• technologies

Never overwhelm recruiters.

Recruiter Mode should exist.

-----------------------------------------------------------

2. Software Engineer

Needs:

• architecture
• implementation
• engineering decisions
• CI/CD
• deployment
• technical depth
• code quality
• scalability

Every project should contain engineering explanations.

-----------------------------------------------------------

3. Hiring Manager / Founder

Needs:

• ownership
• curiosity
• learning ability
• product thinking
• communication
• engineering maturity

The portfolio should communicate not only
what I built,
but WHY I built it.

===========================================================
DESIGN PHILOSOPHY
===========================================================

The portfolio should feel like software built by a company.

NOT

a flashy portfolio.

NOT

a template.

NOT

a clone.

NOT

a landing page.

The experience should feel comparable to products created
by companies known for exceptional software craftsmanship.

Inspiration should come from qualities such as:

• simplicity
• clarity
• confidence
• polish
• subtle motion
• thoughtful interactions
• excellent typography
• spacing
• visual hierarchy
• responsiveness

Never imitate another company's branding or visual identity.

Create an original design that reflects my own engineering style.

===========================================================
MOTION PHILOSOPHY
===========================================================

Motion should explain.

Motion should guide.

Motion should delight.

Motion should never distract.

Every animation answers:

"What is happening?"

NOT

"Look what CSS can do."

===========================================================
ANIMATION RULES
===========================================================

Do NOT animate:

everything.

Animate:

meaning.

Examples:

Cards reveal progressively.

Architecture diagrams build themselves.

Pipelines flow.

Connections animate.

Logs stream.

Nodes activate.

Data moves.

Avoid:

bounce

rubber band

spin forever

long loading animations
